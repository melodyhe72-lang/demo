📚 规则库管理
-- 条正式规则
正式规则库保存在本机浏览器 localStorage（bodyReview_rules），不再写死在代码里；候选规则单独存放在 bodyReview_candidateRules。
导入时会校验格式（数组、每条含 id/title/coreTriggers/optionalTriggers），格式不对会提示错误，不会覆盖现有规则库。
"恢复默认规则"会用代码里的默认规则库重置整个规则库，此操作不可撤销。

🏷 标签规则管理（TAG_RULES）
-- 条标签规则
每条标签规则包含 id / tag / matchFields / keywords / conditions / source / enabled。
matchFields 决定去哪些字段里找关键词（如 meals、exercise_text、subjective_note）；conditions 是结构化判断（如"食欲评分 ≤3"）。同一个 tag 可以有多条规则，命中任意一条就打上标签（相当于"或"关系）。
保存在 bodyReview_tagRules，extractTags() 只负责执行，不再写死关键词。
📐 预测修正规则管理（PREDICTION_MODIFIERS）
-- 条预测修正规则
每条修正规则包含 id / triggers / adjustMin / adjustMax / reason / enabled。当 record 的标签同时包含 triggers 里的所有标签时生效，对预测区间做加减，并把 reason 加入预测原因列表。
保存在 bodyReview_predictionModifiers，predictWeightChange() 只负责按顺序应用这些修正项，不再写死"高钠+便秘""司美格鲁肽+食欲下降"这些判断。
